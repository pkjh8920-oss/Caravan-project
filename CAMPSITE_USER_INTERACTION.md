# 🏕️ 사용자 인터랙션 흐름도 (User Flow)

## 1. 게스트 여정 (Guest Journey)
카라반을 예약하려는 사용자의 핵심 시나리오입니다.

1. **접속 (Landing)**: 메인 페이지에서 추천 카라반 목록 탐색
2. **검색 (Search)**: 지역, 가격대별 필터링
3. **상세 조회 (Detail)**: 편의시설 확인, 지도 위치 파악, 호스트 신뢰도 확인
4. **예약 요청 (Booking Request)**:
   - 체크인/체크아웃 날짜 선택
   - '예약 요청하기' 클릭
   - 상태 변경: `PENDING` (승인 대기)
5. **결제 (Payment)**:
   - 호스트 승인 알림 수신
   - '결제하기' 클릭 (가상 PG 모듈 연동)
   - 상태 변경: `PAID` (예약 확정)

## 2. 호스트 여정 (Host Journey)
자산을 관리하는 공급자의 시나리오입니다.

1. **등록 (Registration)**: 호스트 회원가입 및 내 카라반 등록 (사진, 위치 등)
2. **대시보드 (Dashboard)**: 들어온 예약 요청 확인
3. **승인/거절 (Approval)**:
   - 게스트 프로필 및 일정 확인
   - '승인' 또는 '거절' 버튼 클릭
4. **관리 (Management)**: 카라반 상태 변경 (사용 가능/정비 중)
